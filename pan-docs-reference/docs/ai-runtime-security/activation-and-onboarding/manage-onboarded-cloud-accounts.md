# Manage Onboarded Cloud Accounts in Strata Cloud Manager



 Manage onboarded cloud accounts in Strata Cloud Manager.



 
 






















 

- 

- [Onboard and Activate a Cloud Account in Strata Cloud Manager](/content/techdocs/en_US/ai-runtime-security/activation-and-onboarding/onboard-and-activate-cloud-account-in-scm.html)

| Where Can I Use This? | What Do I Need? |
| --- | --- |
| Manage Cloud Accounts |  |



 After you successfully onboard the cloud accounts, use the Strata Cloud Manager
 (cloud icon) to manage the accounts.

 

1.  Log in to [Strata Cloud Manager](https://stratacloudmanager.paloaltonetworks.com/).
2.  Navigate to AI Security AI Runtime  AI Runtime Firewall.
3.  Click the **Cloud Account Manager** (cloud icon).
4.  Enabling sync configuration settings from Strata Cloud Manager to the Prisma AIRS AI Runtime or VM-Series
 firewall in the cloud account, while disabling stops the sync.
  1.  To **disable** a cloud account, click the pause icon 
 
 

 .
  2.  To **enable** a disabled cloud account, click the enable icon 
 
 

 

5.  Select the **Edit** icon to edit an onboarded cloud account & regenerate
 the Terraform template.
6. 
 
 
 

 
 
7.  Click **Add Cloud Account** to onboard a new cloud account and follow the
 cloud-specific onboarding steps. For details, see how to [Onboard and Activate a Cloud Account in Strata Cloud Manager](/content/techdocs/en_US/ai-runtime-security/activation-and-onboarding/onboard-and-activate-cloud-account-in-scm.html).
