Strata Logging Service empowers most of the threat detection and
 response features in Palo Alto Networks products. Strata Logging Service enables
 you to ingest, store, and forward log data from multiple products. 

## License Support for Qualifying and Non-Qualifying Users

 Starting October 2024, Strata Logging Service supports a new
 licensing scheme. By default, the new Strata Logging Service license comes
 with a one year log retention period. The new license will be available to all
 qualifying users (users who activate Strata Logging Service after October
 2024 and do not currently have an existing Strata Logging Service
 license).

Non-qualifying users, defined as users who have a Strata Logging Service
 license obtained before October 2024, can maintain their current licensing scheme
 until they are migrated to the new licensing model. Starting from March 2025, users
 will receive notifications seven days before the migration process starts. The
 migration will happen automatically and there is no action required from users. 

To use Strata Logging Service, you can either procure a Strata Logging Service license or a Strata Cloud Manager Pro license, which
 includes Strata Logging Service. 

Here are the available options for using Strata Logging Service with different
 products. If you are managing your Palo Alto Networks products using Strata Cloud Manager, it is essential to familiarize yourself with the [Strata Cloud Manager licenses](https://docs.paloaltonetworks.com/strata-cloud-manager/activation-and-onboarding/strata-cloud-manager-licenses-and-support) to better
 understand the license options provided below.

## Migrate to New Strata Logging Service License

 Starting from March 2025, all the users with existing Strata Logging Service
 license will be automatically migrated to the new licensing model that comes with
 unlimited log storage and a one year log retention period. The migration keeps your
 current entitlements and allows you to benefit from new features such as longer log
 retention. You will receive a notification seven days before the migration process
 starts and after the migration is complete. If you are managing your Palo Alto
 Networks products using Strata Cloud Manager, see the [migration paths](https://docs.paloaltonetworks.com/strata-cloud-manager/activation-and-onboarding/strata-cloud-manager-licenses-and-support/license-migration) to know the license
 changes after migration. The migration happens per Customer Support Portal (CSP).
 The table outlines the migration exclusions. 
























 

| What will not be Migrated |
| --- |
| Strata Logging Service with sized storage tenants
                                    hosted in China and FedRamp regions. |
| Any Strata Logging Service tenants with evaluation
                                    licenses only. |
| Strata Logging Service with sized storage evaluation
                                    or expired licenses and Strata Logging Service with sized
                                    storage active license with any product evaluation license ( for
                                    example, Prisma Access, ADEM, AIOps for NGFW) |
